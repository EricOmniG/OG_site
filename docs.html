Overview

Omnigenius
API Reference

The Omnigenius API gives you programmatic access to the Chimera model — deep reasoning, live web search, and streaming responses — via a simple REST interface compatible with any HTTP client.

All requests are made to https://api.omnigenius.co/v1 and return JSON. Authentication is handled via Bearer tokens in the Authorization header.

⚡
New to the API? Jump to Quickstart for a working example in under 2 minutes.
🤖
This documentation is structured for human readers, AI agents, and web crawlers. An OpenAPI 3.1 spec is available at /docs/openapi.json for automated tooling and LLM function-calling.
Getting Started

Quickstart

Make your first API call in three steps.

1. Get an API key

Sign up at omnigenius.co, go to Settings → API Keys, and create a key. Keys are prefixed og- and shown once — store it securely.

2. Send a request

Send a POST to /v1/chat with your message in the request body. The model responds with a JSON object containing the completion.

3. Handle the response

The choices[0].message.content field contains the text response. For streaming, set "stream": true and consume Server-Sent Events instead.

⚠
Never expose API keys in client-side code. Always make API calls from your backend server. Use environment variables — never hardcode keys.

Limits

Rate Limits

Rate limits are applied per API key. Limits are returned in response headers on every request.

PlanRequests/minTokens/day
Free1050,000
Pro605,000,000
Advanced20010,000,000
EnterpriseCustomUnlimited

When you exceed rate limits, the API returns 429 Too Many Requests. The Retry-After header indicates when you can retry.

Endpoints

Chat Completions

Send a conversation to the Chimera model and receive a completion. Supports multi-turn conversations via the messages array.

POST /v1/chat

Request body

ParameterTypeDescription
messagesarrayArray of {role, content} objects. Roles: system, user, assistantRequired
modelstringModel ID. Default: chimera-1Optional
modestringauto | chat | search | reasonOptional
max_tokensintegerMaximum tokens to generate. Default: 2048Optional
temperaturefloat0.0–2.0. Higher = more creative. Default: 0.7Optional
streambooleanIf true, returns SSE stream. Default: falseOptional
200 Success
"id": "chatcmpl-01HXYZ",
"object": "chat.completion",
"model": "chimera-1",
"usage": {
  "prompt_tokens": 24,
  "completion_tokens": 183,
  "total_tokens": 207
},
"choices": [{
  "message": {
    "role": "assistant",
    "content": "The Chimera architecture..."
  },
  "finish_reason": "stop"

Endpoints

Search

Ground the model's response in live web results. The model retrieves sources, ranks them, and synthesises a cited answer. Sources are returned alongside the completion.

POST /v1/search
ParameterTypeDescription
querystringThe search query or full questionRequired
num_sourcesintegerNumber of sources to retrieve. Default: 5, max: 20Optional
freshnessstringday | week | month | anyOptional
synthesisebooleanGenerate an AI summary of results. Default: trueOptional
Endpoints

Reasoning New

Activate the Chimera deep reasoning engine. The model performs multiple internal reasoning loops before responding — ideal for complex analysis, multi-step logic, and research tasks.

POST /v1/reason
ParameterTypeDescription
messagesarrayConversation history as {role, content}Required
depthintegerReasoning loops: 1–8. Higher = more thorough but slower. Default: 3Optional
show_thinkingbooleanInclude intermediate reasoning steps in response. Default: falseOptional
streambooleanStream thinking steps and final answer via SSEOptional
💡
Reasoning mode consumes more tokens than standard chat. Use depth: 1–2 for most tasks. Reserve depth: 6–8 for research-grade problems.
Endpoints

Streaming (SSE)

Set "stream": true on any chat or reasoning request to receive a Server-Sent Events stream. Each event contains a delta chunk. The stream terminates with data: [DONE].

SSE /v1/chat (stream: true)

Event format

Stream chunk
// Each SSE event looks like this:
data: {
  "id": "chatcmpl-01HXYZ",
  "object": "chat.completion.chunk",
  "delta": {
    "role": "assistant",
    "content": "The "
  }
}

data: { "delta": { "content": "Chimera " } }
data: { "delta": { "content": "architecture..." } }
data: [DONE]
Endpoints

Webhooks

Register a URL to receive POST notifications when events occur in your Omnigenius account — completions, errors, usage thresholds, and billing events.

POST /v1/webhooks
Event typeDescription
completion.doneA chat or reasoning completion finished
usage.thresholdToken usage crossed 80% or 100% of plan limit
key.createdA new API key was created
key.revokedAn API key was revoked
billing.paymentA subscription payment succeeded or failed

All webhook payloads include a X-Omnigenius-Signature header — an HMAC-SHA256 signature of the payload using your webhook secret. Always verify this before processing.
